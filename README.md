# Personal Website - Version 3

## Introduction
Here we are again, starting from scratch in the ever-shifting sands of web development.

My website has always been a personal playground - a messy drawer of Lego bricks, full of half broken projects I haphazardly chucked in there. I want to transform it into something more useful and share-worthy. A place for my notes, digital art, and ever-evolving thoughts on tech. I also have grand plans for some tutorials too. I want to start putting some of those Lego projects on display (both figuratively and literally) because what's the point in paying for web hosting if I'm not going to build anything useful with it?

My journey with web development has been a winding one, from hand-crafted HTML pages peppered with PHP to a brief stint with WordPress, and more recently, wrestling with the limitations of Pico CMS. Each step has been a learning experience, sometimes frustrating but always enlightening.

Now, I'm diving headfirst into uncharted waters by building my own CMS. It's not just about breaking free from the constraints of existing systems; it's about challenging myself, digging deeper into the realms of back-end programming, and creating something tailored to my ever-changing digital canvas.

This project is not just for me. I'm designing this new site with flexibility in mind, aiming to make it a useful tool for others as well. And since it's all happening in the public eye on GitHub, this'll be a new leap into web app security.

So, welcome to my project! As they say, the best way to learn is to build. Or, as Richard Feynman is often quoted as saying

> What I cannot create, I do not understand.

Let's see where this goes!

## Vision and Goals
- Build a website that allows for easy editing and maintenance.
- Implement a template system for standard content types like blogs and galleries.
- Design and develop custom pages for unique content, such as SPAs, resumes, and portfolio pieces.

## Tech Stack (Tentative)
- HTML, CSS, JavaScript: For front-end design and interactivity.
- PHP: For server-side scripting.
- Maybe also Node.js and others as needed.
